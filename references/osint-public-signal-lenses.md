# OSINT And Public Signal Lenses

Use when the user asks for public-signal research, company behavior, geopolitical context, public records, hiring/activity signals, ecosystem movement, or non-obvious evidence.

## Public Signal Sources

Use only public, lawful, read-only sources:

- official registries, filings, sanctions lists, procurement records, court/regulatory documents
- company pages, press releases, job postings, patents, standards activity
- GitHub repos, issues, releases, package registries, docs, changelogs
- public social platforms, forums, Reddit, HN, X, YouTube captions, conference talks
- maps, shipping, import/export, satellite, or geospatial sources only when public and appropriate

## Signal Interpretation

Classify every signal:

- confirmed fact
- weak signal
- repeated pattern
- anomaly
- contradiction
- lead for further verification

## Guardrails

- Do not deanonymize private individuals.
- Do not pursue credentials, private accounts, hidden systems, or bypasses.
- Do not use leaked, stolen, or access-controlled material.
- Do not present public chatter as verified fact.
- Keep provenance and uncertainty visible.

## Output Checks

- Explain why each signal matters.
- Note whether independent sources support it.
- Separate official narrative from public behavior.
- Identify next verification steps.
