# Technology Lenses

Use for technical, developer, platform, infrastructure, AI, security, and adoption research.

## Technical Maturity

Assess:

- Production readiness
- Benchmarks and evaluation quality
- Reliability, latency, scalability, and cost
- Security and privacy posture
- Ecosystem support
- Integration burden
- Migration complexity
- Operational failure modes

## Evidence Sources

Prefer:

- Official docs
- Changelogs
- GitHub repos, issues, PRs, releases
- Benchmarks with methodology
- Standards/specifications
- Incident reports
- Developer forum complaints
- Real customer implementations

## Adoption Signals

Look for:

- Stars, forks, contributors, release cadence
- Enterprise references
- Package downloads
- Job postings
- Integration ecosystem
- Community questions and unresolved issues
- Migration guides and compatibility notes

Keep vanity metrics separate from durable adoption.
