# Wild Signal Playbook

Use wild signals to understand market reality, not to establish facts by themselves.

## Sources

- Reddit, X, HN, forums, reviews, GitHub issues, Discord/Slack archives only when public, YouTube captions, comments, newsletters, podcasts, operator blogs.

## What To Extract

- Repeated pain points
- Workarounds and substitutes
- Purchase objections
- Language users use to describe the problem
- Adoption enthusiasm or skepticism
- Complaints about incumbents
- Security/privacy concerns
- Developer friction
- Niche communities forming around a behavior
- Early signs that official narratives are wrong

## Classification

Label every wild signal:

- `Signal`: observed public chatter or behavior
- `Anecdote`: single example
- `Pattern`: repeated across independent sources
- `Contradiction`: conflicts with official/primary evidence
- `Lead`: source or query worth deeper verification

## Rules

- Do not treat social content as representative without evidence.
- Separate sentiment from fact.
- Prefer repeated patterns across independent communities.
- Verify business-critical claims against primary or higher-tier sources.
