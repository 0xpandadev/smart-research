# Research Framing And Hypothesis Engine

Before searching deeply, infer what the user likely wants to understand. Use hypotheses to guide search, not to force a conclusion.

## Reframe The Ask

Identify:

- User intent: understand, compare, verify, monitor, map, investigate, diligence, summarize, find sources
- Time horizon: immediate, quarter, year, multi-year
- Geography and segment
- Required confidence
- Likely output format

If missing, use explicit working assumptions and proceed.

## Build The Issue Tree

Select relevant branches:

- Market: size, growth, segmentation, cyclicality, timing
- Customer: pain, budget, alternatives, adoption, retention, workflow
- Competition: direct, indirect, substitutes, status quo, internal build, services
- Product/technology: maturity, stack, integrations, defensibility, benchmarks
- Business model: pricing, margins, distribution, sales cycle, unit economics
- Regulation/risk: legal, compliance, security, privacy, safety, geopolitics
- Capital/strategy: funding, M&A, valuation, control points, winner/loser dynamics
- Weak signals: complaints, communities, job posts, GitHub activity, social chatter

## Hypothesis Format

Write hypotheses as testable claims:

- "If X is true, we should observe Y in primary sources."
- "If the official narrative is overstated, we should see Z in customer/community evidence."
- "The main bottleneck may be A rather than B."

For each hypothesis, define evidence that would confirm, weaken, or disprove it.

Keep hypotheses provisional. Replace them with source-backed findings as evidence accumulates.
