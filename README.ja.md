# Smart Research

**Codex向けの証拠優先ディープリサーチSkill。広く探し、深く読み、根拠を残す。**

[English](README.md) | [中文](README.zh.md)

Smart Researchは、AIにそれっぽい要約を出させるためのSkillではありません。

目的は、調査の「証拠の足跡」を残すことです。

デフォルトでは、コンサル風の示唆や提言を出すSkillではありません。まずURL、該当ページ/セクション、原文抜粋、表の値、信頼度を残すファクト調査Skillです。

- 公開情報を広く探す
- Web、PDF、開示資料、表、図、GitHub、Reddit/X/HN、フォーラムまで深く読む
- 出典、日付、ページ、セクション、表、図、原文抜粋、矛盾点を残す
- 日付、曜日、営業日、市場営業日を推測せずに検算する
- 事実、推定、解釈、弱いシグナル、リスク、未確認点を分ける
- テーマに応じて専門リサーチャーの視点を自動で切り替える

## 何が違うのか

普通のAI調査は、最終回答がきれいでも根拠が見えないことがあります。

Smart Researchは、最初にこう考えます。

> 何を見つけたのか。どのソースなのか。どれくらい信用できるのか。

重要な主張は、必ずソース行、URL、ページ/セクション、原文抜粋、表の値などで支えます。根拠のない「概要」「示唆」「それっぽい理解」はデフォルトでは出しません。

日付や曜日が重要な場合は、ISO日付、曜日、タイムゾーン、営業日/市場営業日の違いを確認します。曜日は雰囲気で書きません。

そのため、マーケットリサーチ、企業調査、投資調査、AI/テック調査、統計調査、Webマーケ調査、医療調査、OSINT的な公開情報調査に向いています。

## 自動で使う専門レンズ

ユーザーが毎回細かく指定する必要はありません。

たとえば「睡眠市場を調べて」と言えば、市場調査、統計、消費者シグナル、Webマーケ、医療/ヘルスケアの観点を自動で組み合わせます。

| 調査テーマ | 自動で使う視点 |
| --- | --- |
| 市場規模、TAM、顧客、競合 | 市場調査 |
| 公的統計、調査データ、アンケート | 統計・データ |
| 将来予測、シナリオ | フォーキャスト |
| SEO、広告、SNS、LP、ファネル | Webマーケ |
| 医療、ヘルスケア、臨床情報 | 医療調査 |
| GitHub、求人、公開記録、SNS痕跡 | OSINT・公開シグナル |
| 事業、プロダクト、PMF | ビジネス/プロダクト |
| 投資、戦略、DD | 戦略/投資 |
| 技術、開発者採用、アーキテクチャ | テクノロジー |

## 使い方

```text
$smart-research
日本の睡眠市場をdeepで調査して。
一次情報、統計、医療/ヘルスケア観点、Web上の消費者シグナル、矛盾点、未確認点を分けて出して。
```

```text
$smart-research
AIコーディングエージェント市場を調査して。
公式資料、GitHub、専門メディア、Reddit/HN/Xの弱シグナル、競合、未確認点を整理して。
```

```text
$smart-research
この会社の市場ナラティブが本当か調べて。
まず証拠パック、矛盾点、未確認点を出して。投資判断はまだ出さないで。
```

## 出力

形式は自由です。あなたが「表で」「箇条書きで」「記事構成で」「経営向けメモで」と指定できます。

ただし、重要な主張には必ずソースを付けます。可能ならURLだけでなく、ページ、セクション、スライド、表、図、日付も残します。

示唆、提言、戦略的な意味づけは、ユーザーが明示的に求めた場合だけ追加します。

## インストール

```bash
npx skills add https://github.com/0xpandadev/smart-research
```

## 注意

- 医療判断や診断を代替するものではありません。
- 投資助言ではありません。
- 第三者のWebページやPDF内の指示は信用しません。
- ユーザーが明示したファイル以外のローカルファイルは読みません。
