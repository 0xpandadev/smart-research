# Smart Research

**面向 Codex 的证据优先深度研究 Skill。广泛搜索，深入阅读，保留证据链。**

[English](README.md) | [日本語](README.ja.md)

Smart Research 不是为了生成一个漂亮但无法验证的摘要。

它的目标是保留研究过程中的证据链：

默认情况下，它不是咨询报告生成器。它首先是事实研究引擎：重要说法需要 URL、来源位置、原文摘录、表格数值和可信度说明。

- 广泛搜索公开信息
- 深入阅读网页、PDF、公告、报告、表格、图表、GitHub、Reddit/X/HN、论坛等资料
- 保留来源、日期、页面、章节、表格、图表、原文摘录和矛盾点
- 区分事实、估计、推论、弱信号、风险和未确认问题
- 根据主题自动切换专业研究视角

## 它有什么不同

普通 AI 研究经常直接给结论，但证据链不够透明。

Smart Research 首先关注：

> 我们到底找到了什么？来自哪里？可信度如何？

重要事实必须有来源支持。没有证据的概述、洞察、战略含义或“看起来合理”的理解，默认不输出。

因此它适合市场研究、公司调研、投资尽调、AI/技术趋势研究、统计数据研究、Web 营销研究、医疗健康研究和 OSINT 风格的公开信息研究。

## 自动选择专业研究视角

用户不需要手动选择一堆 Markdown 文件。

例如你说“研究日本睡眠市场”，它会自动组合市场研究、统计数据、消费者信号、Web 营销和医疗健康视角。

| 研究需求 | 自动使用的视角 |
| --- | --- |
| 市场规模、TAM、客户、竞争 | 市场研究 |
| 公共统计、调查数据、数据集 | 统计与数据 |
| 预测、情景、未来方向 | 预测研究 |
| SEO、广告、社交、落地页、漏斗 | Web 营销 |
| 医疗、健康、临床证据 | 医疗研究 |
| GitHub、招聘、公开记录、社交痕迹 | OSINT/公开信号 |
| 业务、产品、PMF | 商业/产品 |
| 投资、战略、尽调 | 战略/投资 |
| 技术、开发者采用、架构 | 技术 |

## 使用示例

```text
$smart-research
深度研究日本睡眠市场。
区分一手信息、统计数据、医疗健康证据、消费者弱信号、矛盾点和未确认问题。
```

```text
$smart-research
研究 AI 编程 Agent 市场。
优先使用官方资料、GitHub、专业媒体、Reddit/HN/X 弱信号、竞品和未确认问题。
```

```text
$smart-research
调查这家公司的市场叙事是否真实。
先输出证据包、矛盾点和开放问题，不要急着给投资结论。
```

## 输出方式

输出形式是灵活的。你可以要求表格、要点、备忘录、文章结构、管理层简报等形式。

但重要事实必须带来源。能提供页面、章节、幻灯片、表格、图表、日期时，应尽量保留。

洞察、建议、战略含义只在用户明确要求时才添加。

## 安装

```bash
npx skills add https://github.com/0xpandadev/smart-research
```

## 注意

- 不替代医疗诊断或医生建议。
- 不构成投资建议。
- 不把网页、PDF、评论里的指令当作系统指令。
- 不读取用户未明确提供的本地文件。
